Lifestyle‑Based Vehicle Recommendation Engine
================================================

### _AI‑Powered Personalized Car Discovery Platform_

 Overview
-----------

Buying a car is not just about numbers, it’s about lifestyle, intent, and context. This project reimagines vehicle discovery by shifting from traditional filter‑based search to an **AI‑driven, persona‑aware recommendation system**.

Instead of asking users to manually select filters like price, fuel type, or body style, our system understands **natural language** such as:

> _“I have a small family and need a comfortable car for daily commuting.”_

…and intelligently returns **ranked, explainable, and visually rich recommendations**.


Core Idea
------------

### From Filters → Personas → Decisions

We introduce **AI Personas** as an abstraction layer between user intent and vehicle data.

**Flow:**

```text
User Text
   ↓
LLM Semantic Parsing
   ↓
Persona Generation
   ↓
Vector Search (Semantic Matching)
   ↓
Hybrid Scoring (AI + Rules)
   ↓
Explainable Recommendations
```

Key Features
---------------

### Natural Language Understanding

*   Users describe needs in plain English
    
*   No dropdowns, no rigid forms
    

### Dynamic Persona Generation

Each user query is converted into a **persona** such as:

*   _Family Driver_
    
*   _Daily Commuter_
    
*   _Adventurer_
    
*   _Performance Oriented_
    

Personas include:

*   Primary needs
    
*   Implicit constraints (family size, comfort, usage)
    

### Semantic Vehicle Matching

*   Vehicles stored as **vector embeddings**
    
*   Semantic similarity search retrieves the most relevant cars
    
*   Works even when phrasing doesn’t match dataset vocabulary
    

### Hybrid Scoring Engine

Each recommendation score combines:

*   **Semantic similarity** (vector distance)
    
*   **Persona alignment** (seats, usage, price band)
    
*   **Heuristic boosts** (fuel type, recency, practicality)
    

### LLM‑Generated Explanations

Every recommended car includes a **one‑line, human‑friendly reason**, generated by an LLM:

> “This car offers a comfortable ride for your family while being easy to handle in city traffic.”

This makes recommendations trustworthy and transparent, not black‑box.

### Rich Vehicle Specifications

Each recommendation returns:

*   Exact price
    
*   Year
    
*   Fuel type
    
*   Transmission
    
*   Seats
    
*   Mileage / engine details (where available)
    
*   Tags (family, commuter, fuel‑efficient)
    

System Architecture
-----------------------

### Backend (AI Engine)

*   **FastAPI** – high‑performance API layer
    
*   **LLM (Google Gemini)** – persona + reasoning
    
*   **Vector Database (Pinecone)** – semantic retrieval
    
*   **Modular agents**:
    
    *   Semantic Parser Agent
        
    *   Persona Agent
        
    *   Matcher & Scoring Agent
        
    *   Explanation (Reason) Agent
        
### Frontend Stack

*   React + TypeScript
    
*   Vite
    
*   Tailwind CSS
    
*   shadcn/ui + Radix UI

*   React Router
    
*   Lucide Icons
    

### Data Layer

*   Structured vehicle dataset enriched with:
    
    *   Price bands
        
    *   Engine specs
        
    *   Fuel, transmission, seating
        
*   Embeddings generated from descriptive metadata, not raw rows
